System Architecture & Method Report

Project: Stylyst Pro

Date: December 2, 2025

1. High-Level Architecture

The system operates as a Client-Side Single Page Application (SPA). It relies on a "Thick Client" architecture where the browser handles logic, retrieval, and orchestration, while the Google Gemini API serves as the intelligence backend.

graph TD User[USER] --> Client[FRONTEND CLIENT React + Vite] Client --> UI[UI Layer Tailwind CSS] Client --> State[State Manager React Hooks] Client --> Logic[Logic Core TypeScript] Client --> LocalStore[LOCAL DATA STORE] LocalStore --> Mock[Mock Inventory constants.ts] LocalStore --> Cache[Browser Cache IndexedDB] Client --> AI[AI SERVICE LAYER geminiService.ts] AI --> T2T[Text-to-Text Gemini 2.5 Flash] AI --> V2T[Vision-to-Text Gemini 2.5 Flash] AI --> TTS[Text-to-Speech Gemini TTS]

2. Core Method: "Vibe Search" Pipeline

This is the primary retrieval and ranking method. It uses a Hybrid Search approach: deterministic keyword filtering followed by probabilistic LLM ranking.

graph TD Start(User Enters Query) --> Step1[STEP 1: CANDIDATE RETRIEVAL] Step1 -->|Input: Query + Inventory| Process1[Tokenize & Score Items] Process1 -->|Output| Candidates[Top 60 Raw Candidates] Candidates --> Step2[STEP 2: SEMANTIC RE-RANKING] Step2 -->|Input: Query + Candidates| Gemini[Gemini LLM 2.5 Flash] Gemini -->|Prompt| Prompt[Role: Stylist, Task: Select Top 6] Gemini --> Step3[STEP 3: RESPONSE GENERATION] Step3 -->|Output| Verdict[Stylist Verdict & Explanations] Verdict --> Step4[STEP 4: PRESENTATION] Step4 --> UI_Render[UI Render Results] Step4 --> TTS_Audio[Text-to-Speech Audio]

3. Core Method: "Studio" (Visual Analysis)

This method utilizes Multimodal AI to analyze uploaded images for style extraction.

graph TD UserUpload(User Uploads Image) --> Step1[STEP 1: PRE-PROCESSING] Step1 --> Base64[Convert to Base64] Base64 --> Step2[STEP 2: VISION ANALYSIS] Step2 --> GeminiVision[Gemini Multimodal] GeminiVision -->|Prompt| VoguePrompt[Vogue Editor System Prompt] GeminiVision --> Step3[STEP 3: DATA PARSING] Step3 --> JSON[JSON Response] JSON --> Vibe[Main Vibe] JSON --> Items[Fashion Items] JSON --> Coords[Screen Coordinates] Coords --> End[Display Interactive Overlay]
